Java编程思想篇 Java编程思想深度理论知识

一、Java编程思想总体概述

思考: Java是一个笨重的语言吗？

开闭原则

OCP - Open Closed Principle

软件、函数、类…
扩展 - 开放的
修改 - 封闭的

面向抽象编程

三大特性: 封装、继承、多态

Interface -> 工厂模式 -> IOC/DI

二、通过案例理解抽象的概念

Awkward
Abstraction
Factory
Reflect

1.单纯的Interface可以统一方法的调用，但是它不能统一对象的实例化
2.面向对象 实例化对象 调用方法 (完成业务逻辑)
3.只有一段代码中没有new的出现，才能保持代码的相对稳定，才能逐步实现OCP
4.一段代码如果要保持稳定，就不应该负责对象的实例化
5.对象实例化是不可能消除的
6.把对象实例化的过程转移到其它的代码片段里
7.稳定性是相对的，将不稳定的部分隔离
8.变化造成了不稳定
9.配置文件的变化是允许的，并不违反OCP

三、IOC、DI与DIP

1.IOC: Inversion of Control 控制反转

控制权: 程序员 -> 用户

积木生产厂家(程序员) => 生产一个个积木
玩家(用户) => 搭建组合积木

2.DI: Dependency Injection 依赖注入

属性注入
构造注入
接口注入
…

3.DIP: Dependency Inversion Principle 依赖倒置

高层模块不应该依赖低层模块，两者都应该依赖抽象
抽象不应该依赖细节
细节应该依赖抽象
